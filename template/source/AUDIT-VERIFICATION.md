# AUDIT-VERIFICATION.md
> Source doc (optional) — verification record. Owned by the Researcher role.
> Separates what you CLAIM about the product (DESIGN-AUDIT.md) from how you VERIFIED it.
>
> WHY THIS EXISTS: an audit is only as trustworthy as its evidence. This file forces each
> audit claim to be checked against the live product, with the exact path you took, what you
> actually observed, and any remaining uncertainty. It's what lets you (and a reviewer) tell
> the difference between "I'm fairly sure" and "I confirmed this on screen." Use it when audit
> accuracy matters — e.g. before locking the brief, or before designing against an assumption.
